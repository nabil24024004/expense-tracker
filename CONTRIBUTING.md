# Contributing to Expense Tracker

Thank you for your interest in contributing to Expense Tracker! Contributions are highly appreciated and help make this project better for everyone.

---

## Code of Conduct

By participating in this project, you agree to treat all contributors with respect, professionalism, and kindness.

---

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please create an Issue on GitHub. Include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- Expected vs. actual behavior.
- System details (Android OS version, device or emulator model).

### 2. Suggesting Features
If you have a feature idea, please open a feature request Issue. Describe:
- The problem this feature would solve.
- The proposed solution or behavior.
- Any potential alternatives or mockups.

### 3. Submitting Pull Requests (PRs)
To submit code changes:
1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Implement your changes**, ensuring you write clean, self-documenting code.
3. **Format your code** according to Kotlin coding standards (e.g. ktlint).
4. **Commit your changes** with clear, concise commit messages.
5. **Push to your fork** and submit a **Pull Request** to the `main` branch.
